## 数据字段说明

### 核心字段

| 字段 | 说明 | 示例值 |
|------|------|--------|
| `type` | 流量类型 | direct, referral, search, social, ai_assistants |
| `target` | 流量来源域名 | google.com, twitter.com |
| `traffic` | 当前流量 | 150000 |
| `prev_traffic` | 上期流量 | 100000 |
| `traffic_diff` | 增长率（-1到无穷，0=无变化） | 0.5 (即50%增长) |
| `traffic_share` | 流量占比 | 0.05 (即5%) |

### 流量类型含义

| 类型 | 含义 | 商业解读 |
|------|------|----------|
| `direct` | 直接访问 | 品牌认知度高 |
| `referral` | 引荐流量 | 生态系统/合作关系健康 |
| `search` | 搜索引擎 | SEO 效果/需求强度 |
| `social` | 社交媒体 | 传播力/病毒系数 |
| `ai_assistants` | AI 助手流量 | 新兴渠道曝光 |

### 增长率解读

```
traffic_diff = 1.0   → 100% 增长（翻倍）
traffic_diff = 0.5   → 50% 增长
traffic_diff = 0     → 无变化
traffic_diff = -0.2  → 20% 下降
```

## 分析维度

### 1. 增长分析

识别高增长的流量来源，寻找快速增长的市场机会。

**关键阈值：**
- `traffic_diff > 100%`: 翻倍增长（爆发）
- `traffic_diff > 50%`: 快速增长（热门）
- `traffic_diff < -20%`: 显著下降（衰退）

**应用场景：** 发现新兴市场、识别衰退赛道、评估市场热度

### 2. 规模分析

按流量规模分层，理解不同规模市场的特征。

| 分层 | 流量范围 | 特征 |
|------|----------|------|
| 超大流量 | > 1,000,000 | 成熟市场，竞争激烈 |
| 大流量 | 500K - 1M | 主流市场 |
| 中等流量 | 100K - 500K | **最佳机会区** |
| 小流量 | 10K - 100K | 早期市场 |
| 微流量 | < 10K | 萌芽期/噪声 |

### 3. AI 工具专项分析

识别关键词：
- 通用：ai, gpt, claude, openai, anthropic
- 图像：midjourney, stable, diffusion
- 助手：chatbot, assistant, copilot
- 具体产品：cursor, lovable, suno, elevenlabs, runway

### 4. 机会发现（甜点区）

**理想特征：**
- 流量：100K - 1M（证明有市场）
- 增长率：> 20%（证明在增长）
- 竞争程度：适中（有进入空间）

## 常见分析场景

### 场景 1: 寻找 AI Tool 创业方向

**用户目标：** 想做一个 AI 工具产品，不知道做什么方向

**分析步骤：**
1. 运行 `ai_tools` 分析，查看当前热门的 AI 产品
2. 筛选增长率 > 50% 的工具，找到快速增长的赛道
3. 选择流量在 10万-50万 的工具（竞争相对较小）
4. 按工具类型分类（编程、设计、音频、视频等）
5. 结合自己的技术能力选择方向

**预期产出：** AI 工具排行榜、高增长列表、细分赛道机会分析

### 场景 2: 竞品分析

**用户目标：** 已有产品方向，想了解竞争对手情况

**分析步骤：**
1. 识别同类竞品（通过关键词或产品名）
2. 对比各竞品的流量规模
3. 分析增长趋势（谁在增长，谁在下降）
4. 查看流量来源类型（学习获客渠道）
5. 评估市场份额分布

**预期产出：** 竞品流量对比表、增长趋势图、市场份额分析

### 场景 3: 市场规模评估

**用户目标：** 评估某个细分市场的规模和潜力

**分析步骤：**
1. 识别该细分市场的所有产品
2. 汇总总流量规模
3. 计算市场增长率
4. 分析市场集中度（是否被头部垄断）
5. 识别增长空间

### 场景 4: 发现新趋势

**用户目标：** 发现正在萌芽的新机会

**分析步骤：**
1. 筛选流量 < 50万但增长率 > 100% 的来源
2. 分析这些高增长来源的共同特征
3. 识别新出现的产品类别
4. 评估趋势的可持续性

### 场景 5: 流量来源策略分析

**用户目标：** 学习成功产品的获客策略

**分析步骤：**
1. 选择目标产品
2. 分析其流量来源分布
3. 识别主要引荐来源
4. 评估不同渠道的表现

## 最佳实践

### 数据质量

- 分析前检查缺失值
- 过滤掉流量很小的来源（< 10K）以获得更清晰的洞察
- 验证数据新鲜度和覆盖时间段

### 分析方法

- 先广泛分析，再深入感兴趣的特定领域
- 结合多种分析获得全面视角
- 始终提供背景和解读，不只是原始数字
- 考虑多个因素：增长、规模、竞争、契合度

### 展示

- 使用可视化让趋势清晰
- 突出可操作的洞察，而不只是数据
- 从数据中提供具体案例
- 包含数据局限性的说明

### 战略思考

| 特征 | 判断 |
|------|------|
| 中等流量 + 高增长 | **最佳机会** |
| 高流量 + 低增长 | 成熟市场，难以竞争 |
| 低流量 + 高增长 | 早期市场，风险较大 |
| 负增长 | 避免或深入理解原因 |

### 市场时机判断

- **进入时机**: 市场有一定规模（10万+流量）且快速增长（50%+）
- **避免时机**: 市场萎缩（负增长）或竞争饱和（流量集中度高）
- **观察信号**: 多个相似工具同时快速增长 = 品类爆发

## 数据局限性

### 注意事项

1. 这是**支付页面**的流量，不是产品首页流量
2. 流量高 ≠ 收入高（需要考虑转化率、客单价）
3. 短期流量波动可能是营销活动导致
4. 某些产品可能使用其他支付方式，未计入 Stripe

### 建议补充数据

- 产品定价信息
- 用户评价和口碑
- 团队背景和融资情况
- 技术难度和壁垒

### 常见误区

| 误区 | 正确做法 |
|------|----------|
| 只看流量规模 | 流量大可能竞争激烈，要结合增长看 |
| 忽视增长趋势 | 静态数据没意义，要看变化 |
| 不考虑自身能力 | 好机会也要评估自己能否做出来 |
| 只看短期数据 | 结合多个时间点看长期趋势 |
| 忽视数据来源 | 这是 Stripe 支付页面流量，不是全部 |
